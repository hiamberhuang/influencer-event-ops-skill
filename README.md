# Creator Event Operations Skill  
## 花 $60k 办一场达人活动：从 WBS 到现场执行的一张项目管理工作簿

这个 skill 适用于：品牌方、平台运营、MCN、创作者社区、内容营销团队，用来从 0 到 1 筹备一场线下达人活动。

它不是单纯的「活动表格」，而是一套项目管理方法：

> 先用 WBS 把控整体 project management，再按活动目的推导 rundown，接着确认场地、名单、预算、分工和彩排，最后用座位表把关系设计落到现场。

---

## 你最终需要产出的东西

一场达人活动至少需要这 7 个工作表：

| 顺序 | 表格 | 作用 |
|---|---|---|
| 01 | WBS 总控表 | 管整体节奏、任务、DDL、负责人、状态和风险 |
| 02 | 活动 Rundown 表 | 定义活动环节、时间、目标、物料、人力和现场风险 |
| 03 | 场地与动线表 | 对比活动/晚宴/酒店场地，记录报价、优缺点和实地勘场结论 |
| 04 | 达人名单与邀请表 | 按数据、画风、关系和品牌匹配度筛选达人，并追踪多轮邀请 |
| 05 | 预算表 | 跟踪预算金额、预测金额、实际金额、付款方式、合同/发票 |
| 06 | 团队分工与彩排表 | 拆签到、引导、接待、主持、颁奖、panel、供应商、交通、晚宴 |
| 07 | 座位表 | 安排会议/晚宴座位，让达人、品牌、平台之间自然产生连接 |

# 中文操作方法

## 0. 先建 WBS：把活动变成一个可管理项目

WBS 是整场活动的主控台。不要一上来就写物料、预算、名单。先回答：

1. 这场活动为什么要办？
2. 最终要产生什么结果？
3. 哪些事情必须在活动前完成？
4. 每件事谁负责？什么时候完成？如果卡住，谁来解决？

### WBS 必填字段

| 字段 | 怎么写 |
|---|---|
| 阶段 | 立项 / Rundown / 场地 / 名单 / 预算 / 分工 / 现场 / 复盘 |
| 工作包 | 比如活动目标、活动结构、供应商询价、第一轮邀请、彩排 |
| 任务 | 一句具体动作，不要写虚词 |
| 输出物/检查标准 | 这件事做到什么程度算完成 |
| DDL | 明确日期，不要写“尽快” |
| 负责人 | 只写一个最终责任人 |
| 协作方 | 供应商、品牌、设计、财务、PR、法务、摄影等 |
| 优先级 | P0 / P1 / P2 |
| 状态 | Not Started / In Progress / Blocked / Done |
| 风险/备注 | 写审批周期、供应商假期、老板确认、天气、场地限制等 |

### 新手最容易犯的错

不要写：

> 对接供应商

要写：

> 3月10日前收齐 3 个活动场地报价，包含容量、LED/音响、搭建限制、服务费、付款方式，并输出 shortlist。

## 1. 确认活动 Rundown：先用活动目的决定活动结构

Rundown 不是「几点做什么」那么简单。它应该来自活动目的。

### 示例

如果活动目的是：

> 维护和品牌长期合作的核心达人关系，让大家有荣誉感、加深交流，并同步品牌明年的大动作。

那活动结构可以设计成：

1. **颁奖典礼**  
   给达人荣誉感，让他们觉得被看见、被认可。

2. **Panel 交流**  
   让达人之间、达人和品牌之间产生更深的行业交流。

3. **高端米其林/精致晚宴**  
   给大家一个自然 social 的场景，加强情感连接。

这三个环节不是为了“丰富流程”，而是分别服务三个目标：

| 环节 | 目的 |
|---|---|
| 颁奖 | 荣誉感、身份感、被认可 |
| Panel | 观点交流、行业共识、品牌方向同步 |
| 晚宴 | 关系深化、自然社交、后续合作机会 |

### Rundown 必填字段

| 字段 | 怎么写 |
|---|---|
| 活动板块 | 活动前 / 入场 / 主会 / 中场 / 转场 / 晚宴 |
| 时间 | 开始时间、结束时间、时长 |
| 环节 | 比如签到、开场、颁奖、Panel、茶歇、合影、晚宴 |
| 这个环节服务什么目标 | 每个环节都必须有目的 |
| 环节说明 | 现场具体会发生什么 |
| 供应商/搭建需求 | 舞台、LED、手麦、提词器、控台、拍照墙、茶歇等 |
| 运营物料 | 签到表、桌卡、手环、奖杯、手卡、乘车卡、抽奖箱等 |
| 人力需求 | 主持、控台、摄影、签到、引导、颁奖、panel 运营等 |
| 负责人 | 现场最终 cue 这个环节的人 |
| 风险点/确认项 | 审稿、设备、动线、超时、嘉宾迟到等 |

### Tips

- 每个环节都要写清楚：**为什么存在、现场发生什么、谁负责让它发生**。
- Panel 不要只写“达人分享”，要提前确认主题、嘉宾、问题、禁区、时长。
- 颁奖一定要提前排练上台动线：谁候场、谁递奖、谁 cue 合影、谁下台。
- 晚宴不是“吃饭”这么简单，要设计座位、破冰、抽奖、桌长和社交触点。

## 2. 确定活动举办地点：供应商可以推荐，但你必须亲自看现场

场地不能只看供应商发来的 PPT、照片和报价。你要亲自到现场走一遍动线。

### 场地确认分三步

#### Step 1：让供应商提供 shortlist

至少让供应商提供：

- 活动场地
- 晚宴场地
- 酒店
- 交通/接驳方案
- 搭建报价
- 设备清单
- 服务费和额外费用

#### Step 2：做横向对比

对比维度包括：

| 维度 | 说明 |
|---|---|
| 地理位置 | 达人到达是否方便，离酒店和晚宴多远 |
| 容量 | 是否适合人数，不要太挤也不要太空 |
| 设备 | LED、投影、音响、麦克风、控台是否齐全 |
| 氛围 | 是否符合品牌/活动调性 |
| 动线 | 下车、签到、入场、上台、茶歇、转场是否顺 |
| 搭建限制 | 能不能搭拍照墙、灯光、指引、水牌 |
| 费用 | 场租、服务费、搭建费、加班费、押金 |
| 风险 | 是否半开放、是否不好找、是否雨天不方便 |

#### Step 3：必须实地勘场

你需要用三种身份走一遍：

1. **普通达人**  
   从下车开始：能不能找到入口？签到是否顺？座位是否好找？

2. **VIP/品牌方**  
   是否有休息区？是否方便上台？是否能被妥善接待？

3. **工作人员**  
   物料能不能运进来？奖杯放哪里？摄影站哪里？控台在哪里？出了问题谁能快速到？

### 必看动线 checklist

- 下车点 / 入口 / 电梯
- 签到台位置
- 拍照墙位置
- 主会场入口
- 舞台和屏幕
- 颁奖候场区
- 茶歇区
- 洗手间
- 摄影摄像机位
- 物料存放区
- 供应商进场路径
- 大巴/商务车上车点
- 晚宴入场动线
- 雨天和迟到预案

### Tips

- 不要相信“现场会有工作人员指引”这种空话。你需要知道谁、在哪、拿什么牌子、几点到位。
- 场地越高级，越要确认搭建限制和音响权限。
- 晚宴场地一定要确认是否可以用麦克风、是否可以抽奖、是否有服务费、是否能调整桌型。

## 3. 确认达人名单：用标准筛选，用轮次推进

达人名单不要只按粉丝量排序。你需要把名单分成 P0 / P1 / P2 / Backup。

### 达人筛选标准

| 维度 | 看什么 | 为什么重要 |
|---|---|---|
| 数据维度 | 粉丝量、均播/曝光、互动率、近期增长、历史合作数据 | 判断影响力和稳定性 |
| 内容画风 | 审美、镜头、叙事、表达、是否符合品牌调性 | 决定现场和后续传播质感 |
| 关系远近 | 是否长期合作、是否核心维护对象、是否战略达人 | 关系维护型活动要考虑情感账户 |
| 品牌/活动匹配 | 是否契合明年大方向、品类、目标人群 | 决定活动后是否能转化合作 |
| 商业安全 | 历史争议、内容边界、舆情风险、品牌安全 | 避免活动被风险达人带偏 |

### 优先级判断

你可以用简单规则：

| 优先级 | 标准 |
|---|---|
| P0 | 数据强 + 画风匹配 + 关系重要 + 品牌高度匹配 |
| P1 | 至少命中 2-3 个关键指标，适合参与 |
| P2 | 有潜力或补充多样性，但不是必请 |
| Backup | 可用于补位，避免拒绝率导致名单不足 |

### 邀请分几轮

#### 1）初步确定邀请名单

先建立 longlist，不要一开始就定死。建议至少准备目标人数的 1.3-1.5 倍名单。

例如要请 50 人，建议 longlist 至少 65-75 人。

#### 2）第一轮邀请：简单确认意向

目标不是一次性确认所有细节，而是先问：

- 这个日期是否有兴趣？
- 是否可能到场？
- 是否需要差旅？
- 是否有大致随行需求？

这轮话术要轻，不要让达人觉得负担很重。

#### 3）第二轮邀请及补位

根据第一轮回复情况：

- 已确认的人进入 confirmed
- 未回复的人继续 follow-up
- 拒绝的人记录原因
- 按优先级从 backup 补位

#### 4）确认名单后正式发出邀请函

正式邀请函要包含：

- 活动名称
- 活动时间
- 活动地点
- 活动亮点
- 适合参加的原因
- 交通住宿安排
- 着装建议
- 是否可带随行
- 活动联系人
- 回复截止时间

### Tips

- 第一轮邀请不要过度正式，先降低达人回复成本。
- 正式邀请函不要发太早，除非场地、时间、核心流程都已经锁定。
- 每个达人都要有 POC，不要多人重复联系同一个达人。
- 名单表要和差旅、座位、手环、抽奖、乘车卡打通。

## 4. 确定预算：有了 rundown、场地、名单，预算才会真正清楚

预算应该分两层：

1. **前期预拆预算**：如果你已经知道总预算，比如 $60k，先按大模块拆。
2. **最终锁定预算**：当名单、场地、环节确认后，预算会更接近真实。

### 推荐预算结构

| 科目 | 说明 |
|---|---|
| 活动场地 | 场租、设备、服务费、押金 |
| 晚宴/餐饮 | 晚宴、酒水、服务费、茶歇 |
| 差旅 | 机票、高铁、打车、接驳车 |
| 酒店 | 达人住宿、VIP 住宿、工作人员住宿 |
| 搭建/制作 | 拍照墙、KV、物料、LED、摄影摄像 |
| 奖杯/物料 | 奖杯、桌卡、手环、姓名贴、指引水牌 |
| 伴手礼/抽奖 | 伴手礼、奖品、礼袋、包装 |
| 人力/供应商 | 主持、摄影、摄像、茶歇、搭建供应商 |
| Buffer | 建议保留 10%-15% |

### 预算表必填字段

| 字段 | 说明 |
|---|---|
| 一级科目 | Venue / Travel / Production / Gift 等 |
| 二级科目 | 活动场地、晚宴、酒店、交通、奖杯等 |
| 预算金额 | 原计划金额 |
| 预测金额 | 当前估算金额 |
| 实际金额 | 已发生金额 |
| 预算差异 | 自动计算是否超支 |
| 付款方式 | 合同、报销、公司卡、B2P 等 |
| 付款状态 | Not Started / In Progress / Paid / Invoiced |
| 经手人 | 谁负责付款或报销 |
| 收款方 | 供应商/餐厅/酒店/达人 |
| 合同/发票/附件 | 链接或文件名 |
| 备注 | 超支原因、审批状态、特殊说明 |

### Tips

- 预算不是财务一个人的事。每个新增需求必须有业务 owner。
- 晚宴一定要确认服务费、酒水、开瓶费、最低消费。
- 差旅预算最容易浮动，要保留 buffer。
- 供应商报价要拆明细，不要只收一个总价。
- 活动结束后 48 小时内更新实际费用，否则后续结算会很乱。

## 5. 团队分工与彩排：把活动当天拆成具体动作

不要写“大家现场配合”。这是不可执行的。

你要把活动当天拆成一个个岗位：

| 岗位 | 负责什么 |
|---|---|
| Project Owner | 对最终结果负责，处理升级问题 |
| Stage Manager | 控整体流程，cue 主持、控台、颁奖、转场 |
| Reception Lead | 负责签到、手环、桌卡、姓名贴 |
| Creator Liaison | 负责达人接待、迟到、特殊需求 |
| VIP/Brand Liaison | 负责品牌/VIP 接待 |
| Award Lead | 负责奖杯顺序、候场、上台、合影 |
| Panel Lead | 负责嘉宾、话题、时长和上台 cue 点 |
| AV/Control Lead | 负责 PPT、视频、音乐、话筒、灯光 |
| Photo/Video Lead | 负责摄影摄像、合影、素材回收 |
| Vendor Lead | 负责供应商到场、搭建、茶歇、设备 |
| Transport Lead | 负责酒店接驳、跟车、晚宴转场 |
| Dinner Lead | 负责晚宴入场、桌长、抽奖、伴手礼 |
| Emergency Owner | 负责舆情、医疗、天气、设备、嘉宾迟到等异常 |

### 彩排必须测什么

- 主持人完整走流程
- PPT、视频、音乐、麦克风测试
- 颁奖上台动线
- Panel 座位和麦克风传递
- 签到发手环/桌卡
- 茶歇区动线
- 合影站位
- 乘车和晚宴转场
- 晚宴抽奖流程
- 雨天/迟到/设备异常预案

### Tips

- 彩排不是开会，是让人真的站到点位上走一遍。
- 所有现场关键岗位都要有 backup。
- 现场不要让一个人同时负责两个高压岗位，比如签到和 VIP 接待。
- 每个岗位都要知道：我什么时候到、我站哪里、我拿什么、我听谁 cue、出问题找谁。

## 6. 座位表：这是关系设计，不是行政表格

座位表的目标不是“把人坐满”，而是设计交流。

### 会议座位

适合优先考虑：

- VIP/品牌/平台负责人坐前排或核心区
- 需要上台的嘉宾坐靠近舞台/过道
- 获奖达人按上台顺序安排，减少动线混乱
- 工作人员坐靠边和靠过道，方便机动

### 晚宴座位

适合优先考虑：

- 每桌 1 位能破冰的人
- 不同内容方向轻混搭
- 熟人不要全部坐一桌
- 品牌方旁边安排关键达人
- 避免有潜在竞争或关系尴尬的人同桌
- 记录忌口、随行、提前离场

### 座位信息要打通

同一个编号最好同时用于：

- 桌卡
- 手环
- 抽奖编号
- 乘车卡
- 签到表
- 工作人员查询表

### Tips

- 现场要准备 3 个版本：签到台版、桌面版、工作人员手机截图版。
- 座位表最终版要给到签到、引导、晚宴、交通、Project Owner。
- 如果有人临时缺席，优先调整晚宴座位，不要轻易改主会座位。

# English Workflow

## 0. Build the WBS first

The WBS is your project control center. It answers:

- What needs to be done?
- Who owns it?
- By when?
- What does “done” mean?
- What is blocked?

Do not write vague tasks like “talk to vendor.” Write actionable tasks like:

> Collect 3 comparable venue quotes by Mar 10, including capacity, AV, setup restrictions, service fee, payment terms, and site visit availability.

## 1. Confirm the rundown based on the event goal

If the goal is to maintain strategic creator relationships and announce next-year brand moves, the event can be structured as:

- Awards: recognition and status
- Panel: deeper exchange and shared direction
- Premium dinner: social bonding and future collaboration

Each segment must explain why it exists, what happens on site, and who owns it.

## 2. Confirm the venue and visit the site yourself

Suppliers can provide options and quotes, but you must check the flow in person:

- guest drop-off
- sign-in desk
- photo wall
- main room
- stage
- awards staging area
- tea break area
- restrooms
- camera positions
- vendor loading path
- transport pick-up point
- dinner entrance
- rain / delay / overtime plan

Walk through the space as a creator, a VIP, and a staff member.

## 3. Build the creator list and invite in rounds

Prioritize creators by:

- performance data
- content style and production quality
- brand/event fit
- relationship depth
- brand safety / controversy risk

Move from longlist → soft invite → second invite/backfill → final invitation.

## 4. Lock the budget

Start with a rough budget split, then lock it after venue, guest list and rundown become stable.

Recommended categories:

- venue
- dinner / catering
- travel
- hotel
- production / build
- materials
- awards / gifts
- vendors
- contingency buffer

## 5. Assign duties and rehearse

Break the event into concrete roles:

- project owner
- stage manager
- reception lead
- creator liaison
- VIP / brand liaison
- award lead
- panel lead
- AV / control lead
- photo / video lead
- vendor lead
- transport lead
- dinner lead
- emergency owner

Rehearsal must be physical. People should stand in position, test the equipment, run the awards flow, and test transitions.

## 6. Design the seating chart

Seating is relationship design.

Use seating to:

- help creators meet useful people
- support brand / VIP conversations
- avoid awkward combinations
- make stage and dinner flows smoother
- connect table cards, wristbands, raffle IDs and transport cards
